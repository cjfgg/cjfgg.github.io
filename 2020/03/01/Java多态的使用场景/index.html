Java多态的使用场景

个人看法：

多态是Java语言的特点，多态即意味着父类引用子类对象的向下转型。使用多态的前提条件是，该类是一个父类。

多态的编译看左，运行看右。意思是程序在未启动前an会被当成是Animal类型的对象，在运行时会被当成是Cat类型的对象，可以使用Cat类中的成员方法。

多态的使用场景1：

当有许多子类时，要想实现方法的功能，就得写多个方法，方法接收的参数不同，就得写一个新的方法，这样就显得代码非常的笨重，所以使用多态就可以让代码变得简洁明朗。

解决方法

用父类型的形参接收数据，这样父类型的形参可以接收其任意子类对象。但是程序在运行时依然会把an当成是子类对象。第一个showAnimal方法，cat是一个Cat类型的对象，父类型的形参接收了cat对象，an此时就是cat。

第二个showAnimal方法，an接收了dog，an此时就变成dog了。
